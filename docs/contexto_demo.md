# Contexto Demo

Este projeto e uma versao publica de demonstracao para portfolio. Ele simula a operacao de uma empresa de engenharia ficticia, com foco em gestao de servicos tecnicos, ordens de servico, atividades, cronogramas, pagamentos e dashboards.

## Objetivo

Demonstrar uma aplicacao web simples e funcional para organizar o fluxo operacional de uma equipe de engenharia, mantendo dados ficticios e sem depender de qualquer contexto real de empresa ou cliente.

## Entidades Principais

- Clientes
- Colaboradores
- Servicos
- Ordens de servico
- Pagamentos
- Atividades
- Cronograma
- Disponibilidade
- Indicadores de performance

## Regras Gerais da Demo

- O banco usado pela versao publica e `data/demo.db`.
- Os dados cadastrados sao ficticios.
- A identidade visual usa o nome generico "Sistema de Gestao de Engenharia".
- O fallback visual da marca e `SGE`.
- Logo e favicon sao opcionais.
- Bancos reais, backups e arquivos `.env` nao devem ser versionados.

## Fluxo Demonstrado

1. Um usuario demo acessa o sistema por login.
2. O dashboard resume clientes, colaboradores, ordens, pagamentos e atividades.
3. O usuario navega pelos cadastros principais.
4. Atividades podem ser organizadas por data, colaborador, cliente, servico e ordem.
5. Relatorios de disponibilidade e performance ajudam a acompanhar a operacao simulada.
